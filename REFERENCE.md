# Program mechanics reference (researched 2026-07-06)

Baked into the calculator's math — re-verify if a program changes its rules.

## PC Optimum (Shoppers Drug Mart, Real Canadian Superstore, Loblaws…)

- 1,000 points = $1 (1 pt = 0.1¢). Minimum redemption 10,000 = $10.
- Base earn at Shoppers: **15 pts/$1** (1.5% back). Superstore base earn is lower/offer-driven — set the base field accordingly.
- **Multiplier events ("20x the points")**: multiply the base → 20x at Shoppers = 300 pts/$1 = **30% back**. Usually gated ("when you spend $50+"); targeted offers can be 25x+ with PC Financial cards.
- **Bonus redemption events ("Spend Your Points" / Super Redemption)**: tiered.
  Typical menu: 50,000 → $75 (+50%) · 130,000 → $200 (+54%) · 250,000 → $400 (+60%) ·
  500,000 → $900 (+80%). Point value jumps from 0.10¢ to 0.15–0.18¢.
  Purchase must be ≥ the discount. Tiers vary per event — enter as printed.

## Triangle Rewards (Canadian Tire, Sport Chek, Mark's…)

- CT Money is dollars; spends like cash. Base earn at Canadian Tire: **0.4%**.
- **Multiplier events ("20x CT Money")**: the bonus is base × N *added to* the base you'd
  normally collect → 20x ≈ **8.4% back** (Triangle's own example: $100 @ 20x = $8 bonus).
- **Redemption events**: "redeem CT Money, get 20% back", typically capped at $1,000
  redeemed. Net effect: $200 of goods costs $160 of CT Money.
- Triangle Bonus Days = multi-banner event windows; spend-and-get thresholds also common
  (use the app's Every-$X / Spend-tiers modes for those).
